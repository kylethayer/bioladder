#Ways you can help BioLadder.Org

# Wiki Content #

If you wish to help with adding wiki content (adding animals/plants/etc.), see the [wiki page](http://bioladder.org/wiki/index.php?title=Help:HowToHelp) on how to help there.

# Programming #

For information on what is being worked on, see:

  * [Design Document](https://bioladder.googlecode.com/hg/resources/View%20design.pdf)
  * [Issues](https://code.google.com/p/bioladder/issues/list)
  * [Google Group](https://groups.google.com/forum/?fromgroups#!forum/bioladderorg) discussions

### Wiki ###

The wiki uses [MediaWiki](http://www.mediawiki.org/) with [Semantic MediaWiki](http://semantic-mediawiki.org/) and [Semantic Forms](http://www.mediawiki.org/wiki/Extension:Semantic_Forms)

### Viewer ###

The viewer uses [Sencha Touch](http://www.sencha.com/products/touch) 2.2, which is built on Javascript and HTML5. It communicates with the wiki using the [Semantic Wiki Api](http://semantic-mediawiki.org/wiki/Ask_API)

### Download and Setup ###

To download the code and run it yourself, go to the [Source Tab](https://code.google.com/p/bioladder/source/checkout) and use the mercurial command to clone the repository, then follow the instructions in the [README file](https://code.google.com/p/bioladder/source/browse/README)

### Contribute Changes ###

If you wish to contribute a change, the best way to do that is to create a clone of the project in the [Source Tab](https://code.google.com/p/bioladder/source/checkout), push your changes to that cloned repository and inform the [Google Group](https://groups.google.com/forum/?fromgroups#!forum/bioladderorg) of the change you made.

If you reliably contribute good changes, you may be added to the committer list of the project and be able to push changes directly to the main repository.
